2. YOUR METRICS PAGE (`src/app/info/page.tsx`) - **CRITICAL UPGRADE AREA**
--------------------------------------------------------------------------------
* IN DETAIL - WHAT IS INCLUDED:
  - Three large visual metrics cards displaying: Current Weight (kg), Height (cm), and Calculated BMI with health status.
  - An update form: Numeric input fields for Weight (kg), Height (cm), Age (years), and selectors for Gender (Male, Female, Other) and Activity Level (Sedentary, Light, Moderate, Active, Very Active).
  - Save button writing data instantly to `localStorage`.

* WHAT IT DOES:
  - Calculates a simple Body Mass Index (BMI) using weight and height:
    $$BMI = \frac{Weight (kg)}{(Height (m))^2}$$
  - Saves measurements for plan setup and target calorie formulas.

* REDUNDANCIES & CONSOLIDATIONS:
  - Redundant: Currently, this page only calculates a basic BMI. Weight and activity levels are also prompted in the plans panel `/plans` and dashboard weight-logs.
  - Fix: Turn this page into the **ultimate scientific baseline command center**. It should house ALL body calculations, making `/plans` and other setup cards purely operational and clean.

* DEEP IMPROVEMENTS - SCIENTIFIC FORMULAS:
  To turn this metrics page into an elite analytics dashboard, calculate these advanced indicators:
  
  A. **Basal Metabolic Rate (BMR)** - Using the scientific Harris-Benedict (revised by Roza and Shizgal) formula:
     - For Men:
       $$BMR = 88.362 + (13.397 \times Weight_{kg}) + (4.799 \times Height_{cm}) - (5.677 \times Age_{years})$$
     - For Women:
       $$BMR = 447.593 + (9.247 \times Weight_{kg}) + (3.098 \times Height_{cm}) - (4.330 \times Age_{years})$$
     
  B. **Total Daily Energy Expenditure (TDEE)** - Calculating real calorie needs based on metabolic multipliers:
     - Sedentary: $BMR \times 1.2$
     - Lightly Active: $BMR \times 1.375$
     - Moderately Active: $BMR \times 1.55$
     - Very Active: $BMR \times 1.725$
     - Super Active: $BMR \times 1.9$
     
  C. **Lean Body Mass (LBM)** - Calculating exact muscle vs. fat weight using the Boer Formula:
     - For Men:
       $$LBM = (0.407 \times Weight_{kg}) + (0.267 \times Height_{cm}) - 19.2$$
     - For Women:
       $$LBM = (0.252 \times Weight_{kg}) + (0.473 \times Height_{cm}) - 48.3$$
     
  D. **Body Fat Percentage Estimate** - Dynamically derived from LBM:
     $$BodyFat\% = \left(\frac{Weight - LBM}{Weight}\right) \times 100$$
     
  E. **Ideal Daily Hydration Base** - Formulated using body weight and activity multiplier:
     $$Water_{ml} = (Weight_{kg} \times 35) + \text{Activity Bonus (500ml for active levels)}$$

  - RENDER PLAN: Add a gorgeous dynamic grid showing these 5 new parameters with clean, HSL-colored visual dials and progress rings. Explain the scientific math under a "Show Science Formulas" toggle!

================================================================================
           PART 5: SECURITY, SPEED, AND PERFORMANCE RECOMMENDATIONS
================================================================================

1. LAZY LOADING INTERACTIVE COMPONENTS (Speed & First Paint)
   - Heavy visual libraries like Chart.js and dynamic maps increase FCP times.
   - Solution: Use Next.js dynamic imports to load charts only when they scroll into viewport:
     ```tsx
     import dynamic from 'next/dynamic';
     const LineChart = dynamic(() => import('react-chartjs-2').then(mod => mod.Line), {
       ssr: false,
       loading: () => <div className="h-80 bg-gray-50 animate-pulse rounded-2xl" />
     });
     ```

2. SECURE SESSION VALIDATION (XSS & CSRF Protection)
   - Problem: Storing auth tokens or personal credentials in `localStorage` makes them susceptible to cross-site scripting (XSS) attacks.
   - Solution: Transition credentials to signed JWT cookies with `HttpOnly` and `Secure` flags, routing calculations securely through middleware API queries.

3. MEMOIZING PERFORMANCE CALCULATIONS
   - Prevent unnecessary calculations on rapid keystrokes or inputs:
     ```tsx
     const computedMacros = useMemo(() => calculateDailyTargets(weight, activity), [weight, activity]);
     ```
